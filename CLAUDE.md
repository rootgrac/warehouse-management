# 仓库管理 (Warehouse Management)

## 项目概述

一个多人共享的仓库管理 Web 应用，支持手机端操作。核心功能：货物入库/出库、库存查询、价格管理。多人共享同一仓库数据，一人操作后其他人的软件实时同步变动。

## 技术选型

| 层面 | 技术 | 原因 |
|------|------|------|
| 前端框架 | **React 18 + Vite** | 生态成熟，移动端适配好 |
| UI 组件库 | **Ant Design Mobile** | 专为手机端设计，中文字体友好 |
| 后端服务 | **Supabase** (BaaS) | 自带数据库 + 实时同步 + 用户认证，省去后端开发 |
| 数据库 | **PostgreSQL** (Supabase 内置) | 可靠的关系型数据库 |
| 部署方式 | **PWA** (可安装到手机桌面) | 无需应用商店，浏览器打开即可用 |
| 实时同步 | **Supabase Realtime** | 数据变更自动推送到所有在线用户 |

> **为什么选 Supabase？** 
> - 自带用户注册/登录系统
> - 内置实时订阅功能（一个用户改数据，其他用户秒级看到变化）
> - 免费额度足够小仓库使用（500MB 数据库，50,000 月活用户）
> - 有 Row-Level Security 保证数据安全
> - 国内可通过阿里云/腾讯云部署 Supabase 替代方案（如 [memfirecloud.com](https://memfirecloud.com) 提供国内 Supabase 兼容服务）

## 核心功能

### 1. 用户系统
- 注册/登录（手机号或邮箱）
- 创建或加入一个"仓库团队"
- 团队内所有人共享库存数据

### 2. 货物管理
- 添加新货物（名称、规格、单位、初始库存、单价）
- 编辑货物信息
- 删除货物（软删除）

### 3. 入库管理
- 选择货物 → 输入数量 → 输入进货价 → 确认入库
- 入库后库存自动增加
- 入库记录可查询

### 4. 出库管理
- 选择货物 → 输入数量 → 确认出库
- 出库后库存自动减少
- 库存不足时提醒
- 出库记录可查询

### 5. 库存总览
- 所有货物的当前库存一览表
- 显示货物名称、规格、当前库存、单价、总价值
- 低库存预警（库存低于设定阈值时标红）

### 6. 流水记录
- 所有入库/出库操作的时间线
- 显示操作人、操作类型、货物、数量、时间
- 可按时间范围筛选

### 7. 实时共享
- 同一团队的成员看到相同数据
- A 操作入库 → B 的手机上库存数字立即更新
- 无需手动刷新

## 数据模型

```
团队 (teams)
  - id: UUID
  - name: 团队名称
  - invite_code: 邀请码（其他人凭此加入）
  - created_at: 创建时间

货物 (products)
  - id: UUID
  - team_id: 所属团队
  - name: 货物名称
  - spec: 规格/型号
  - unit: 单位（个/箱/kg/米...）
  - current_stock: 当前库存数量
  - unit_price: 当前单价（元）
  - low_stock_threshold: 低库存预警阈值
  - created_at / updated_at

入库记录 (stock_in_records)
  - id: UUID
  - product_id: 货物
  - team_id: 团队
  - quantity: 入库数量
  - unit_price: 进货单价
  - total_amount: 总金额
  - operator: 操作人 (user_id)
  - remark: 备注
  - created_at

出库记录 (stock_out_records)
  - id: UUID
  - product_id: 货物
  - team_id: 团队
  - quantity: 出库数量
  - operator: 操作人 (user_id)
  - remark: 备注
  - created_at
```

## 页面结构（手机端）

```
底部导航栏
├── 📦 库存     → 货物列表 + 搜索 + 低库存标记
├── 📥 入库     → 快速入库表单
├── 📤 出库     → 快速出库表单  
└── 👤 我的     → 个人信息 + 团队管理 + 邀请成员 + 流水记录
```

## 实时同步机制

```
用户A操作入库 → Supabase写入数据库
                        ↓
          Supabase Realtime 检测到数据变更
                        ↓
          广播给同一 team_id 的所有在线用户
                        ↓
          用户B/C/D 的界面自动更新库存数字
```

## 共享流程

1. 张三注册并创建团队"1号仓库"，获得邀请码 `ABC123`
2. 张三把邀请码发给李四
3. 李四注册后输入邀请码，加入"1号仓库"
4. 两人看到相同的库存数据，任何一方的操作都会同步给对方
